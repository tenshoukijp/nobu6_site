nullptr

  • 概要

    ここでは、C++/CLIの着目すべき文法のうち、nullptrを中心に解説しています。

  • nullptr

    C++11においてポインタが何もさしていないときnullptrを使うように、
    マネージドヒープに確保したハンドルが何もさしていないときもnullptrを使います。

    初期化してないハンドルはnullptrに設定されています。

    nullptrは０ではありません。C++11のnullptrが０ではないの同様です。

    ハンドルがbool値のような扱いを受けるときはnullptrとの比較が行われるのとおなじ意味になります。


    C++/CLIのnullptr 
    String^ s;
if(s)  // if ( s != nullptr )と同じ   ...
bool isnull = !s;  // bool isnull = s ==nullptr; と同じ

  • ネイティブC++11のnullptr

    ネイティブのnullptrは、いかなるオブジェクトも指さないポインタを表します。
    あらゆるポインタ型に代入できますが、組み込み型への代入はできません。

    C++11のnullptr 
        int *pi = nullptr;
    double* pd = nullptr;
    int i = nullptr;     // エラー: iはポインタではない

    からポインタは、各ポインタ型ごとに存在するのではなく、ただ１つのnullptrが、あらゆるポインタ型で利用されます。
